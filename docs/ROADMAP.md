# GlazePad 路线图

## 0.1.x 维护线

目标：保持 `0.1.0` 首个公开预览版本稳定可用，只处理阻断安装、启动、剪贴板、持久化和发布流程的问题。

- 修复安装包、启动或退出相关问题。
- 修复文本和图片剪贴板读写问题。
- 修复本地状态保存、恢复和图片清理问题。
- 修复托盘、隐藏唤醒、全局快捷键的明显异常。
- 保持 Windows NSIS 安装包发布流程稳定。

原则：`0.1.x` 不引入大型功能和明显改变使用方式的交互调整。

## 0.2.0 方向

目标：补齐小工具长期使用时最需要的基础能力，让 GlazePad 从公开预览走向更稳定的日常可用版本。

当前已纳入 `0.2.0`：

- `+` 单击新增空白文本 Tab，右键收纳当前剪贴板。
- 新建 Tab 标题复用已删除的最小空缺编号。
- 托盘菜单支持配色、四档透明度和开机自启动，并显示当前勾选状态。
- 复制失败显示具体错误。
- 关于版本号自动读取应用版本。

### 启动与更新

- 评估自动更新方案，优先考虑 Tauri updater；如果暂不接入自动更新，先提供新版本提示或清晰的手动更新说明。

### 快捷键

- 允许用户修改全局快捷键，避免 `Alt + Space` 与系统或其他工具冲突。
- 检测快捷键注册失败，并在界面或托盘中给出可理解的提示。

### 数据管理

- 提供清空数据入口。
- 提供导出和备份方案，至少覆盖 `state.json` 和 `images`。
- 保留“打开数据目录”，并在文档中明确数据文件用途。

### 首次使用体验

- 优化首次启动时的默认内容和状态提示。
- 明确隐藏到屏幕边缘、托盘常驻和快捷键唤醒之间的关系。
- 检查多显示器、不同 DPI 和任务栏位置下的窗口定位表现。

### 发布工程

- 处理 GitHub Actions 中 Node.js runtime 弃用警告。
- 记录代码签名方案、费用和 CI 接入方式，等公开传播规模扩大后再执行。
- 明确发布后验证清单，覆盖安装、覆盖安装、卸载、启动、托盘、快捷键、剪贴板和本地数据。

## 暂不计划

- 完整剪贴板历史。
- 账号、云同步和跨设备同步。
- 复杂笔记编辑器。
- 多平台发布，除非 Windows 版本已经稳定并且确有需求。
