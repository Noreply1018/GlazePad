# GlazePad 交互设计

本文档以当前 Open Design 原型 `index.html` 为准。GlazePad 现在不是剪贴板历史、分类收藏夹或侧边栏列表，而是一个贴在桌面上的透明短浮窗：每个 Tab 是一个独立暂存槽，每个槽位只保存一份内容。

## 交互原则

GlazePad 的交互目标是低干扰、低摩擦、稳定可预期。

- 用户主动把剪贴板内容收进 GlazePad，不自动记录全部剪贴板历史。
- 当前只展示一个槽位，减少扫描负担。
- 操作入口集中在顶部，内容区保持安静。
- 文本和图片可以有不同视觉形态，但不能改变窗口高度。
- 所有反馈优先写入顶部状态文字，不使用 toast、弹窗或原生 tooltip。

## 基础形态

GlazePad 是一个固定高度的透明桌面浮窗。

- 默认宽度接近 386px。
- 默认高度固定，文本和图片 Tab 切换时不允许外框跳动。
- 浮窗背景使用半透明玻璃质感，适合停留在桌面或其他窗口上方。
- 主体区域分为顶部操作区、Tab 行、当前槽位、底部状态行。

内容区必须稳定：

- 文本内容超出时在槽位内部滚动。
- 图片内容在同一个固定槽位内铺满展示。
- 图片使用 `object-fit: cover`，允许轻微裁切。
- 图片尺寸和类型显示在底部状态行，而不是压在图片内部。

## Tab 模型

Tab 不是分类。Tab 是独立暂存槽。

默认提供 4 个 Tab：

- Tab 1
- Tab 2
- Tab 3
- Tab 4

每个 Tab 下方只有一个内容槽位。用户切换 Tab 时，只是在切换当前显示的暂存槽，而不是进入某个分类列表。

交互要求：

- Tab 按钮使用固定宽度，切换时不能发生左右位移。
- 当前 Tab 只通过背景、颜色和边框变化表达激活态，不改变字重、尺寸或 padding。
- Tab 超出时允许横向滚动。
- 新增 Tab 后立即切换到新 Tab。
- 删除 Tab 后切换到邻近 Tab。
- 至少保留一个 Tab。

当前版本不支持 Tab 改名。后续如果要恢复改名，应单独设计，不要挤占当前顶部操作区。

## 顶部操作

顶部操作区从左到右为：

1. 复制当前槽位
2. 删除当前 Tab
3. 新增 Tab 并收纳剪贴板
4. 隐藏 GlazePad

四个按钮必须保持同尺寸、同状态风格：

- 使用图标，不写文字。
- 保留 `aria-label`，不要使用 `title`，避免浏览器原生提示。
- 复制按钮使用蓝绿色动作色。
- 删除按钮使用低饱和危险色。
- 新增按钮使用普通前景色。
- 隐藏按钮使用弱化灰色。

复制行为：

- 文本槽位：把当前文本写入系统剪贴板。
- 图片槽位：把当前图片写入系统剪贴板。
- 点击复制时不能全选文本，不能改变当前编辑选择状态。
- 复制失败只更新顶部状态文字。

删除行为：

- 删除当前 Tab。
- 如果只剩一个 Tab，不删除，并在顶部状态文字提示。
- 删除后优先切到右侧邻近 Tab；没有右侧时切到左侧。

新增行为：

- 点击 `+` 后立即读取系统剪贴板。
- 优先判断图片，其次判断文本。
- 如果剪贴板是图片，创建图片槽位。
- 如果剪贴板是文本，创建文本槽位。
- 如果浏览器原型无法读取真实剪贴板，可以用示例内容兜底；正式版应使用系统剪贴板 API。

隐藏行为：

- 点击 `›` 后，浮窗向右侧收起。
- 收起速度保持较快，避免拖沓。
- 隐藏后只保留右侧极细唤醒区域。

## 文本槽位

文本槽位是一个单一透明内容框，不允许出现“框套框”。

交互要求：

- 直接在当前槽位内编辑文本。
- 输入后自动保存到本地状态。
- 长文本在槽位内部滚动，不改变浮窗高度。
- 代码、命令或明显技术文本可以切换为等宽字体。
- 点击复制按钮时只复制，不聚焦并全选文本。

当前版本不提供单独清空按钮。用户可以直接编辑或删除当前 Tab。

## 图片槽位

图片槽位是轻薄照片贴片，不是图库、相册或卡片列表。

视觉和行为要求：

- 图片几乎无框，让图片本身成为主体。
- 图片铺满固定槽位，可轻微裁切。
- 图片不应该撑高浮窗。
- 不显示文件名和文件大小。
- 底部状态行显示极简信息，例如 `PNG · 1024×768`。
- 图片槽位不支持编辑。
- 复制按钮复制当前图片。

图片与文本切换时，外框、Tab 行、底部状态行的位置都必须稳定。

## 底部状态行

底部状态行分为左侧内容状态和右侧模式标识。

左侧显示：

- 当前暂存槽数量。
- 当前槽位类型：文本或图片。
- 图片槽位时显示图片类型和尺寸。

示例：

- `4 个暂存槽 · 文本`
- `4 个暂存槽 · 图片  PNG · 1024×768`

右侧显示：

- `透明桌面模式`

底部信息只做轻量提示，不承担主要操作。

## 隐藏与唤醒

GlazePad 只有显示态和隐藏态，不做 mini mode。

显示态：

- 完整透明短浮窗。
- 可以操作 Tab、复制、删除、新增和编辑当前槽位。

隐藏态：

- 浮窗收起到屏幕右侧。
- 只保留一条细唤醒边。
- 点击唤醒边或触发快捷键后恢复显示态。

快捷键建议：

- `Alt + Space`：显示 / 隐藏切换。
- `Esc`：显示态下隐藏。

正式 Tauri 版本应使用全局快捷键实现，HTML 原型中的键盘监听只作为交互演示。

## 动效规范

动效服务于状态确认，不做装饰。

隐藏 / 唤醒：

- 使用右侧磁吸感。
- 收起较快，保持干脆。
- 弹出稍慢，带明显但顺滑的拉力和回弹。
- 点击唤醒后，右侧唤醒边必须立刻消失，不能贴在屏幕边缘残留。
- 动画只控制 `transform` 和少量 `opacity`，避免 `blur/filter` 参与主运动。
- 动画应可取消，连续点击不能卡住状态。

Tab 切换：

- 内容槽可以有短促的换槽反馈。
- Tab 按钮自身不能位移。
- 不要使用大幅滑动或复杂转场。

按钮：

- 悬停轻微上移。
- 按下轻微缩放。
- 所有顶部按钮手感一致。

无障碍与系统偏好：

- 遵守 `prefers-reduced-motion`。
- 系统开启减弱动态时，禁用大幅弹性动效。

## 不做的交互

当前版本明确不做：

- 剪贴板历史列表。
- Tab 分类下的多卡片列表。
- Tab 改名。
- 当前槽位清空按钮。
- toast 消息弹窗。
- 浏览器原生 `title` 提示。
- 点击复制时全选文本。
- 复杂 mini mode。
- 图片图库、拖拽排序或图片编辑。

这些限制是为了让 GlazePad 保持“透明桌面暂存槽”的轻量定位。

## 实现注意事项

正式实现时建议拆成以下交互模块：

- `GlazePadShell`：透明浮窗、固定尺寸、隐藏/唤醒。
- `SlotTabs`：Tab 列表、新增、删除、激活态。
- `SlotContent`：文本槽位和图片槽位。
- `ClipboardBridge`：读取/写入系统剪贴板。
- `MotionController`：可取消的隐藏/唤醒动画。
- `StorageAdapter`：本地持久化。

状态模型应围绕 slot，而不是 category：

```ts
type Slot =
  | {
      id: string
      title: string
      type: "text"
      content: string
    }
  | {
      id: string
      title: string
      type: "image"
      imageUrl: string
      imageType: string
      width: number
      height: number
    };
```

每次渲染只显示 `activeSlot`，不要把所有内容作为列表渲染出来。
