# GlazePad 交互设计

## 总体原则

GlazePad 的交互目标是低摩擦、低干扰、可控。

用户主动决定哪些剪贴板内容进入 GlazePad。应用不自动记录所有复制历史，避免内容污染和隐私风险。

## 唤醒与隐藏

第一版采用键盘优先：

- 通过全局快捷键唤醒侧边栏。
- 再次触发快捷键或按 `Esc` 隐藏侧边栏。
- 侧边栏显示时保持置顶。
- 侧边栏默认吸附在屏幕右侧。

第二阶段再考虑混合流：

- 鼠标移动到屏幕右侧边缘时自动滑出。
- 鼠标离开侧边栏区域后自动缩回。

## Tab 管理

默认提供 4 个 Tab，可在实现时使用以下初始命名：

- 常用话术
- 代码片段
- 临时素材
- 备注

交互要求：

- 支持新增 Tab。
- 支持重命名 Tab。
- 支持删除 Tab。
- 不设置硬性数量限制，但界面应鼓励保持 3 到 5 个主要 Tab。
- 当前 Tab 下的卡片按添加时间倒序排列，最新内容在最上方。

## 智能加号

智能加号是 GlazePad 的核心入口。

触发方式：

- 点击当前 Tab 附近的加号按钮。
- 后续可增加全局快捷键，用于直接把当前剪贴板内容收入当前 Tab。

触发后的行为：

1. 读取系统剪贴板。
2. 判断内容类型。
3. 如果是文本，创建文本卡片。
4. 如果是图片，创建图片卡片。
5. 新卡片插入当前 Tab 顶部。
6. 给出轻量成功反馈。

如果剪贴板为空或类型暂不支持，应给出轻量提示，不弹出复杂对话框。

## 卡片交互

卡片是内容的基本单位。

文本卡片：

- 常态显示文本预览。
- 鼠标悬浮时显示复制和删除按钮。
- 点击复制按钮，将文本写入系统剪贴板。
- 点击文本主体，进入原地编辑状态。
- 编辑完成后自动保存，或通过明确保存动作保存。

图片卡片：

- 常态显示图片缩略图。
- 鼠标悬浮时显示复制和删除按钮。
- 点击复制按钮，将图片写入系统剪贴板。
- 第一版不做图片编辑。

## 编辑与复制分离

卡片主体点击不直接复制。

这样设计是为了保留编辑能力，避免用户想修改内容时误触复制。复制必须通过明确的复制按钮触发。

## 清空

每个 Tab 提供清空当前 Tab 的入口。

清空是破坏性操作，应至少提供一次确认，或者提供短时间撤销能力。第一版可以优先采用确认弹窗，后续再做撤销。

## 尺寸调整

第一版支持手动调整侧边栏宽度。

建议约束：

- 最小宽度保证 Tab、按钮、卡片内容可用。
- 最大宽度不超过屏幕宽度的一定比例。
- 宽度设置本地保存。

## 视觉方向

- 高级浅色。
- 简洁白框。
- 轻量阴影和边框。
- 适度半透明或磨砂质感。
- 图标按钮克制，不使用大面积高饱和色。
- 信息密度适中，避免做成厚重笔记软件。
