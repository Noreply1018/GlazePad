# 路线图

本文档只记录尚未完成的方向。已经发布或已经实现的内容写进 `CHANGELOG.md`，发布检查写进 `docs/release.md`。

## 启动与更新

- 评估自动更新方案，优先考虑 Tauri updater；如果暂不接入自动更新，先提供新版本提示或清晰的手动更新说明。

## 快捷键

- 允许用户修改全局快捷键，避免 `Alt + Space` 与系统或其他工具冲突。
- 检测快捷键注册失败，并在界面或托盘中给出可理解的提示。

## 数据管理

- 提供清空数据入口。
- 提供导出和备份方案，至少覆盖 `state.json` 和 `images`。
- 保留“打开数据目录”，并在文档中明确数据文件用途。

## 首次使用体验

- 优化首次启动时的默认内容和状态提示。
- 明确隐藏到屏幕边缘、托盘常驻和快捷键唤醒之间的关系。
- 检查多显示器、不同 DPI 和任务栏位置下的窗口定位表现。

## 发布工程

- 处理 GitHub Actions 中 Node.js runtime 弃用警告。
- 记录代码签名方案、费用和 CI 接入方式，等公开传播规模扩大后再执行。

## 暂不计划

- 完整剪贴板历史。
- 账号、云同步和跨设备同步。
- 复杂笔记编辑器。
- 多平台发布，除非 Windows 版本已经稳定并且确有需求。
