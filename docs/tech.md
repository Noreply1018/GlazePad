# GlazePad 技术方案

## 开发环境

GlazePad 主要在 WSL 中开发和管理：

```text
/home/lh/projects/GlazePad
```

代码可以在 WSL 内编辑，但最终应用仍依赖 Windows 桌面能力运行。涉及窗口置顶、全局快捷键、系统剪贴板、右侧吸附和桌面透明效果时，应在 Windows 原生环境验证。

## 推荐技术栈

- Tauri
- React
- TypeScript
- Vite
- pnpm

选择理由：

- 适合做轻量桌面暂存工具。
- 相比 Electron，Tauri 体积和内存占用更适合长期常驻。
- React + TypeScript 便于维护 Tab 槽位、类型判断、隐藏状态和设置界面。
- Vite + pnpm 提供快的本地开发体验。

## 当前原型约束

现阶段的 UI 原型已经明确，不再是旧版侧边栏或多卡片分类面板，而是：

- 透明短浮窗。
- 四个独立 Tab 槽位。
- 每个 Tab 只对应一个内容框。
- 内容类型只有 `text` 和 `image` 两类。
- 顶部只保留复制、删除、新增、隐藏四个动作。
- 隐藏后从右侧边缘磁吸唤醒。
- 整体高度固定，切换文本和图片不会改变外框高度。

## 建议的数据模型

第一版建议使用本地持久化，不做云同步。

```ts
type Slot =
  | {
      id: string;
      title: string;
      type: "text";
      content: string;
    }
  | {
      id: string;
      title: string;
      type: "image";
      imageUrl: string;
      imageType: string;
      width: number;
      height: number;
    };

type AppState = {
  activeId: string;
  hidden: boolean;
  slots: Slot[];
};
```

图片建议只保存本地路径或临时资源引用，不要长期转成 base64。

## 当前实现重点

### 1. 固定高度短浮窗

- 外框高度固定。
- 文本和图片都塞进同一个固定内容槽。
- 文本内容多时内部滚动。
- 图片使用 `cover` 或 `contain` 按设计决定，但不能撑高外框。

### 2. Tab 槽位

- 每个 Tab 是一个独立暂存槽。
- `+` 新增槽位时先判断剪贴板类型。
- `−` 删除当前槽位，但至少保留一个。
- 切换 Tab 只改变当前槽位内容，不改变窗口高度。

### 3. 文本与图片两种槽位

- 文本槽位：透明便签感，允许编辑，复制当前文本即可。
- 图片槽位：轻薄照片贴片感，几乎无框，图片本身是主体。
- 图片信息只保留极简类型和尺寸，放在底部状态行右侧。

### 4. 顶部操作

- 复制
- 删除
- 新增
- 隐藏

这些按钮应保持统一尺寸和玻璃质感，不要让新增按钮比其它按钮更亮。

### 5. 隐藏与唤醒

- 右侧点击唤醒边在隐藏态可见。
- 点击唤醒后立即消失。
- 唤醒采用磁吸展开，有明显拉力和回弹，但不要出现中途卡断。
- 收起速度保持稳定，唤醒速度可以更从容。

## 状态与交互

- 复制、新增、删除、隐藏的反馈统一走顶部状态文字。
- 不使用 toast。
- 不使用原生 `title` tooltip。
- 不提供改名和清空功能。
- 不提供多卡片列表。
- 不提供分类式 Tab 语义。

## 本地存储

建议先用本地 JSON 或轻量存储记录：

- Tab 顺序
- 当前激活 Tab
- 文本内容
- 图片路径或临时引用
- 窗口隐藏状态

如果后续需要搜索、标签、迁移，再切到 SQLite。

## Windows 能力验证

下面这些能力需要在 Windows 原生环境验证：

- 全局快捷键注册
- 系统剪贴板读写
- 图片剪贴板读写
- 窗口置顶
- 右侧边缘吸附
- 隐藏 / 唤醒动画
- 高 DPI 缩放

## 开发优先级

### P0

- 初始化 Tauri 项目。
- 复刻当前透明短浮窗。
- 实现 Tab 槽位模型。
- 支持文本 / 图片两种内容。
- 支持隐藏 / 唤醒。
- 支持本地持久化。

### P1

- 剪贴板自动识别类型。
- 图片剪贴板支持。
- 复制当前槽位内容。
- 删除当前 Tab。
- 边缘唤醒交互。

### P2

- 更稳定的磁吸动画。
- 右侧吸附优化。
- 高 DPI 与多显示器适配。
- 更细的状态保存与恢复。
